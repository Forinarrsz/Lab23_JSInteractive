# Проект: Интерактивный калькулятор

## 1. Описание проекта
Данный проект представляет собой веб-приложение для выполнения базовых арифметических расчетов. Программа разработана с использованием объектно-ориентированного подхода на языке JavaScript, что обеспечивает модульность и легкость расширения функционала.

## 2. Функциональность
*   **Поддерживаемые операции:** Сложение (+), вычитание (-), умножение (*), деление (/).
*   **Ограничения:** 
    *   Запрещен ввод нескольких точек в одном числе.
    *   Блокировка последовательного ввода двух и более операторов (например, нельзя ввести `5++2`).
    *   Невозможно начать выражение с оператора (кроме унарного минуса).
*   **Обработка ошибок:** При возникновении исключительных ситуаций (некорректный синтаксис или деление на ноль) калькулятор выводит на экран надпись «Ошибка» и блокирует ввод до нажатия кнопки «C».

## 3. Архитектура
### Структура файлов
*   `index.html` — HTML-разметка интерфейса.
*   `style.css` — Визуальное оформление (сетка кнопок и адаптивность).
*   `calculator.js` — Файл с описанием логики (класс `Calculator`).
*   `main.js` — Точка входа для инициализации приложения.

### Класс Calculator
Класс является "мозгом" приложения. Основные свойства:
*   `expression`: текущая строка для вычисления.
*   `display`: ссылка на элемент экрана.

Основные методы:
*   `init()`: привязывает обработчики событий к кнопкам.
*   `calculate()`: запускает процесс вычисления.
*   `clear()`: сбрасывает состояние.

### Схема взаимодействия
`Пользователь (клик)` —> `Event Listener` —> `Метод класса Calculator` —> `Валидация` —> `Обновление DOM (дисплея)`.

## 4. Логика работы
*   **Последовательность при нажатии кнопок:** 
    1. Перехват текстового значения кнопки.
    2. Проверка: является ли кнопка служебной (`C` или `=`).
    3. Если кнопка обычная — попытка добавить символ в очередь.
*   **Валидация ввода:** Перед добавлением символа в строку проверяется последний символ текущего выражения. Если пользователь пытается ввести две точки или два оператора подряд, ввод игнорируется.
*   **Вычисление:** Происходит при нажатии на кнопку `=`. Строка `expression` обрабатывается встроенным интерпретатором внутри блока `try...catch`. В случае успеха результат выводится на экран, в случае провала — выводится ошибка.
